toc-rail demo

Reading rail for long articles

Build a fixed outline from article headings and show reading progress beside the page.

  • Vanilla TypeScript, ESM-only, zero runtime dependencies
  • Import the CSS and call mountTocRail()
  • Show the rail on wide screens and keep mobile focused on the article

Install

Import the package and the default CSS.

Import

import { mountTocRail } from "toc-rail";
import "toc-rail/style.css";

mountTocRail({
  content: "article",
  headings: "article h2[id], article h3[id]"
});

Content requirements

Headings used by the rail should have stable id values.

Scroll behavior

The progress line and active link update as the reader moves through the article.

Active state

The active item receives is-active, data-toc-rail-active="true", and aria-current="location".

Anchor links

Rail links are normal fragment anchors. Use scroll-margin-top for fixed headers.

Styling

Change color, width, and placement with CSS variables.

.toc-rail {
  --toc-rail-accent: #197ca8;
  --toc-rail-width: 184px;
  --toc-rail-right: 2rem;
  --toc-rail-top: max(96px, 18vh);
}

Before and after content

By default, the rail hides before the article starts and after it ends.

Bottom fade

The rail fades near the end of the article so it does not hang beside footer content.

Progress only

Pass headings: false when you only need reading progress.

mountTocRail({
  content: "article",
  headings: false,
  title: false
});

CSS-first styling

JavaScript manages DOM, active state, and progress variables. Visual design stays in CSS.

Responsive behavior

The rail hides automatically below minWidth.

Narrow screens

This demo hides the right rail below 1140px.

Lifecycle

mountTocRail() returns a handle for route changes and dynamic content.

const rail = mountTocRail({ content: "article" });

rail.refresh();
rail.update();
rail.unmount();

refresh()

Call after headings or article height change.

unmount()

Call before leaving the page to remove listeners, timers, frames, and DOM.

Final check

Check rail navigation, active state, progress, and narrow-screen hiding before publishing.
